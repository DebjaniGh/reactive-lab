How does this work? @if (panel.name === 'Subject') {

A plain Subject has no memory. When a value is emitted, only current subscribers receive it. Late subscribers start with an empty slate — they only see values emitted after they subscribe.

} @else if (panel.name === 'BehaviorSubject') {

BehaviorSubject always holds the most recent value. It requires an initial value at creation time. When a new subscriber joins, it immediately receives the current value — then all future emissions. This is the backbone of Angular service-based state management.

} @else {

ReplaySubject(n) buffers the last n values. When a new subscriber joins, those buffered values are immediately replayed — then all future emissions follow. Unlike BehaviorSubject, it doesn't require an initial value and can replay more than one past value.
