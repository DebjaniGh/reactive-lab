How does this work?

Without shareReplay(), every call to .subscribe() creates a new, independent execution of the Observable. Each execution triggers a separate HTTP request — even if the data is identical.

How does this work?

shareReplay(1) uses a ReplaySubject(1) internally. The first subscriber triggers the HTTP call. The result is cached. All subsequent subscribers receive the cached result immediately — no new HTTP call is made. This is identical to how ReplaySubject replays the last value to late subscribers — which you already saw in the Subjects demo.
