⚠️ Each | async creates a subscription. Forgetting cleanup creates zombie subscriptions.

How does this work?

BehaviorSubject is push-based. You call .next(value) to push new values to all subscribers. The template uses | async to subscribe and auto-unsubscribe. Derived state uses .pipe(map()). Each pipe in the template creates a separate subscription that Angular manages through the async pipe.

Subscription Management

Subscriptions: 0 ⚡

No | async. No .subscribe(). No cleanup needed. Angular reads signals directly.

✅ Zero chance of zombie subscriptions.

How does this work?

Signal is pull-based. You read it by calling count() — like calling a function. Angular tracks which templates depend on which signals. When a signal changes, Angular updates only the affected views. Derived state uses computed() — no pipe(), no map(), no subscriptions.
